# Blog Post Generation Prompt

以下のルールに従って、ポートフォリオサイトのブログ記事を1本生成してください。

## ルール

1. `src/content/blog/` に新しい Markdown ファイルを作成する
2. ファイル名は英語のケバブケース（例: `my-article-title.md`）
3. 既存の記事と重複しないテーマを選ぶ
4. frontmatter は以下のフォーマットに従う:

```yaml
---
title: "記事タイトル（日本語）"
description: "1-2文の概要（日本語）"
date: YYYY-MM-DD  # 今日の日付
tags: ["タグ1", "タグ2"]
draft: false
---
```

## テーマの方向性

以下のカテゴリからローテーションで選ぶ（既存記事を確認して重複を避ける）：

- **技術**: Astro, TypeScript, Python, Flutter, AI/ML, データ分析, クラウド, セキュリティ
- **キャリア**: SE→コンサルのキャリアパス, 技術選定の考え方, プロジェクトマネジメント
- **開発プラクティス**: テスト, CI/CD, コードレビュー, リファクタリング, 設計パターン
- **学習**: 資格取得, 新技術のキャッチアップ方法, アウトプット駆動の学習

## 記事の品質基準

- 800〜1500字程度
- 実体験や具体例を交えた内容（一般論だけにならない）
- h2, h3 で構造化
- 結論や学びで締める
- 過度にAI臭い文章は避ける（「〜と言えるでしょう」の連発等）

## 手順

1. `src/content/blog/` の既存記事を確認
2. 重複しないテーマを決定
3. Markdown ファイルを作成
4. ビルドが通ることを確認（`npm run build`）
